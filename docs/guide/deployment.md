# 部署说明

当前这个 Task 2 脚手架已经可以产出静态站点文件，构建结果位于 `docs/.vitepress/dist`。

这意味着当前最实际的部署方式，是将该目录交给任意静态文件托管环境来提供访问。

## 本地 Docker 运行

这是后续可补齐的一条集成路径：为当前静态构建结果增加 Docker 运行时文件，并通过 Compose 启动一个静态站点服务。

## 接入宿主机 Nginx

这是另一条计划中的集成路径：当本地容器运行入口补齐后，再由宿主机 Nginx 将二级域名反向代理到该静态站点服务。

## 远程一键部署

这是面向后续维护的扩展方向：在运行时文件稳定后，再补充一键部署入口，用于同步项目并触发远端更新。
